存储过程在 服务器 端对数据库中的数据进行操作，并将结果返回到 客户 端

存储过程的参数分为 输入参数和输出参数 两种类型

在一般情况下，系统存储过程返回0表示 执行成功 返回非0表示 执行失败

触发器与存储过程不同，触发器由 事件 触发，存储过程是由 对象 执行

用户自定义函数与存储过程的区别：

  • 存储过程通过OUTPUT参数返回数据，用户自定义函数通过返回值返回数据
  • 存储过程可以独立执行，用户自定义函数通常出现在SELECT语句中
  • 存储过程具有较为复杂的功能，用户自定义函数的功能通常具有针对性

存储过程由输入/输出参数、编程语句和返回值组成

用户自定义函数由参数、编程语句和返回值组成

chapter7

完整性包括 列级完整性、行级完整性、引用完整性 3中类型

约束包括 主键约束、唯一性约束、检查约束、默认约束、外键约束、非NULL约束 6种类型

使用 sp_bindefault 存储过程绑定默认值

使用 sp_unbindrule 存储过程解除绑定规则

默认值对象与 默认约束 完成相同的功能，规则对象与 检查约束 完成相同的功能

约束对象(规则)与检查约束的异同点？

  • 相同点：规则可以实现与检查约束相同功能的完整性约束。
  • 不同点：
    • 规则是一个数据对象，它可以独立存在。当表删除时，与之绑定的规则不会删除。而表删除时，表中的检查约束也会删除。
    • 一个规则可以与表中的多个列进行多次绑定，而一个检查约束只能针对表一个列。
    • 规则不仅可以对列进行约束，还可以对用户自定义的数据类型进行约束。而检查约束只能对列进行约束。
    • 规则不能应用于已经插入到表中的数据。而约束可以对已插入到表中的数据进行检查。

默认对象与默认约束的异同点？ 

  • 相同点：默认对象可以实现与默认约束相同功能的完整性约束。
  • 不同点：
    • 默认对象是一个数据对象，它可以独立存在。当表删除时，与之绑定的规则不会删除。而表删除时，表中的默认约束也会删除。
    • 一个默认对象可以与表中的多个列进行多次绑定，而一个默认约束只能针对表一个列。
    • 默认对象不仅可以对列进行约束，还可以对用户自定义的数据类型进行约束。而默认约束只能对列进行约束。不能为已有默认绑定的列或用户定义数据类型创建DEFAULT约束。

chapter5

使用 create table 语句可以创建表

SQL Server数据库的表由 元组和属性 组成

使用 delete 语句可以快速地删除表中的所有数据

SQL Server的表约束包括：

  • 每个数据库可以创建多少个表，没有限制
  • 每个表中最多定义1024列
  • 表和列的命名要遵守标识符命名规则
  • 列名在各自的表中必须唯一
  • 每列必须有一种并且只能有一种数据类型

having 子句的功能是指定组或聚合的搜索条件。having 子句通常与 group by 子句一起使用

表结构所包含的内容：表的名称，表的列，列的数据类型，列的空值，列的主键，列的索引，列的约束，列的默认值

delete from table 和 drop table 的区别：

  • delete table 删除表的数据，表仍然存在，仍可以对表进行操作
  • drop table，是从数据库中删除表。删除后，表结构不在存在，无法再对该表进行任何操作
通配符 功能
% 包含零个或者多个任意字符的字符串
_ 包含单个任意字符

having 与 where 的区别：

  • having 子句与 where 子句的相同点是二者都为查询数据指定查询条件，可达到同样的查询结果

  • where 子句必须在 group by 子句之前，having 子句必须在 group by 子句之后

  • where 子句中条件表达式不能包含聚合函数，having 子句中条件表达式可以包含聚合函数

chapter4

在create database语句中使用 attach 关键字附加数据库

使用 drop database 删除数据库

使用存储过程 sp_detach_db 分离数据库

根据多存储信息的不同，数据库文件可以分为 主数据库文件，次数据库文件和事务日志文件

名称 用途 扩展名 数量
主数据库文件 存放数据库对象 .mdf 每个数据库只有1个
次数据库文件 数据较多时补充存放数据库对象 .ndf 可以没有，也可有多个
事务日志文件 存放数据库事务日志信息 .ldf 至少1个，也可由多个

分离与删除数据库的区别：

  • 相同：两者都是从数据库管理系统中除去数据库，在管理界面中再也看不到该数据库
  • 不同：删除数据库时数据库的物理文件被彻底删除，分离数据库时数据库的物理文件依然存在，所以才能被复制，然后在另一台服务器上被附加

chapter1

计算机数据处理经历了四个阶段：人工管理阶段、文件管理阶段、数据库管理阶段、分布式数据库管理阶段

数据模型按照应用层次分为三种类型：概念模型、逻辑模型、物理模型

关系完整性约束是保证关系模型中数据完整性的重要手段，包括 实体完整性，参照完整性，和用户定义完整性 三种类型

概念结构设计的任务是 将需求分析阶段得到的用户需求抽象为概念模型

逻辑结构设计的任务是 将E-R图转化为特定数据管理系统支持的逻辑模型

物理结构设计的任务是 为逻辑模型选取一个最合适的物理结构，包括数据库在物理设备上的存储结构和存取方法

数据库系统的组成 数据库系统由数据库、支持数据库运行的硬件和软件、数据库管理系统、数据库应用程序和人员组成

数据库管理系统应具备的功能 

  • 数据定义：数据定义是数据库的定义功能
  • 数据操作：数据操作是数据库的操作功能
  • 数据查询：数据查询是数据库的检索功能
  • 数据控制：数据控制是数据库的保护功能

数据库设计的几个阶段

需求分析，概念结构设计，逻辑结构设计，物理结构设计，数据库实施，数据库的运行和维护
